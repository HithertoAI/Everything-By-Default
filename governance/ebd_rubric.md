The EbD Innovation Rubric: The Three-Question Gatekeeper
To maintain the Holokleric Standard, every new piece of technology, vendor, or process must pass this rubric. If the answer to any question is "No," the technology is rejected or moved back to The Commons for remediation.

Is it Secure by Default (Zero Trust)?

Does the data processing happen at the edge (local SoC)?

Is there a "Zero-Persistence" state-wipe after each transaction?

Is it Accessible by Default (Total Agency)?

Can the most sensory-deprived or cognitively-taxed user navigate the interface with 100% efficacy?

Does it eliminate the "Accessibility Tax" (extra steps/cost for inclusion)?

Is it Efficient by Default (Unified Ontology)?

Does the tool speak the "Standard Language" of the existing platform, or is it a "Science Experiment" requiring a custom bridge?

Does it reduce the user's cognitive load rather than adding a new "Setting" to manage?
